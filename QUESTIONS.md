### 2026-05-08 — Phase 6: tsx dev script doesn't emit decorator metadata

| #   | Question                                                                                       | Working assumption                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --- | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 20  | NestJS DI silently breaks in `pnpm --filter @mnela/<app> dev` (tsx watch). Same for api + mcp. | Pre-existing project-wide limitation: tsx → esbuild drops `design:paramtypes` metadata, so `@Injectable` constructor parameters land as `undefined`. The vitest config at `apps/api/vitest.config.ts:5-8` already calls this out and uses `unplugin-swc` — that's why tests pass. Production via `tsc` build also works (compiled JS keeps the metadata). Phase 6 smoke went through `pnpm --filter @mnela/mcp build` + `node dist/main.js`. **Workaround for next phase:** swap the per-app `dev` script for `node --import=@swc-node/register/esm-register src/main.ts` (or `tsc -w` + `node --watch dist/main.js`), or accept the dev/dist split and document. Not blocking Phase 6 acceptance because the integration tests pass under SWC and the production container will use `tsc` build. Resolve in Phase 11 polish. |

## How to use

When you hit something ambiguous in the TZ that isn't a hard blocker, write a question here, make a reasonable assumption to keep moving, and surface it to the user at the next natural checkpoint.

If it _is_ a hard blocker, stop and ask before proceeding.
